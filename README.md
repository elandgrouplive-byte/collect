# 채널 콘텐츠 발행 비교 도구

특정 키워드의 콘텐츠가 여러 채널에서 같은 시기에 얼마나 발행됐는지 확인하는 도구.
구글 디스커버 노출량과 주제 발행량의 상관관계를 검증하는 용도.

## 모니터링 채널

| 채널 | 자동 수집 가능성 |
|---|---|
| 아던트뉴스 생활·문화 | ✅ 안정적 |
| 위키트리 라이프 | ⚠️ 동적 페이지, 일부 글만 잡힐 수 있음 |
| 헬슈오 (다음) | ⚠️ 카카오 차단 가능성 |

## 배포 방법

### 1. GitHub 저장소 생성

1. GitHub에서 새 저장소 만들기 (예: `channel-monitor`)
2. **Public**으로 설정 (private이면 GitHub Actions 무료 한도가 있음)

### 2. 파일 업로드

이 폴더의 모든 파일을 저장소에 push.

```
channel-monitor/
├── .github/workflows/collect.yml
├── scrapers/
│   ├── __init__.py
│   ├── ardentnews.py
│   ├── wikitree.py
│   └── daum_channels.py
├── collect.py
├── data.json
├── index.html
└── README.md
```

### 3. GitHub Pages 활성화

1. 저장소 Settings → Pages
2. Source: `Deploy from a branch`
3. Branch: `main` / `/ (root)`
4. 1~2분 후 `https://<username>.github.io/channel-monitor/` 로 접속

### 4. 첫 데이터 수집 실행

자동 실행은 매일 한국 시간 자정. 첫 실행을 수동으로 트리거하려면:

1. 저장소 Actions 탭
2. `Collect Channel Articles` 워크플로우 선택
3. `Run workflow` 클릭
4. 5~10분 후 완료. `data.json`이 자동 커밋됨.

## 사용 방법

1. GitHub Pages URL 접속
2. **우리 콘텐츠 핵심 키워드** 입력 (예: "밥솥 그릭요거트")
3. **우리 콘텐츠 발행일** 선택
4. **전/후 일수** 조정 (기본 ±14일)
5. **발행 현황 분석** 클릭

결과:
- 채널별 매칭 건수
- 발행 시그널 강도 (강함/중간/약함)
- 매칭된 글 제목과 링크

## 데이터 누적

- 매일 자동으로 새 글 수집
- 60일 이상 된 데이터는 자동 삭제 (저장소 용량 관리)
- 처음 한 달은 데이터 부족할 수 있음

## 트러블슈팅

### data.json이 비어있음
GitHub Actions가 한 번도 실행되지 않았거나 모든 채널 수집이 실패한 경우.
Actions 탭에서 로그 확인.

### 다음 채널만 0건이 계속 나옴
카카오가 GitHub Actions IP를 차단했을 가능성. 다음 채널은 자동화가 어려운 영역.
이 경우 다음 채널은 직접 모니터링하셔야 함.

### 위키트리만 0건
위키트리는 동적 로딩 페이지라 일부 글만 잡힘. 정상 동작이지만 누락 가능성 있음.
